## Paper's claim — concrete params + model topology

**Source:** `2603.01499v2.pdf` Table 4 page 15 ("Ablative studies for internal state protection against ISA").

| Applied mechanism | AttnScore TTRSR(%)↓ | HiddenState TTRSR(%)↓ |
|---|---:|---:|
| Noise | 87.14 | 40.0 |
| Noise + KeyMat | **87.14** | 0.82 |
| Noise + KeyMat + Head&BlockPerm | **0.0** | 0.0 |

**Hyperparameters used by paper** (§7.1 page 13):
- Expansion size `h = 128`
- Embedding noise `α_e = 1.0`
- Head noise `α_h = 0.2`
- Block permutation window `β = 8`
- Block permutation sampling `γ = 10³`
- Matrix coefficient `λ = 0.3`
- Generation temp = 0.65, top-k = 20, top-p = 0.95

**Model topology** (paper Table 4 doesn't specify model, but the chapter context + Table 2 immediately preceding suggests Qwen2.5-14B-Instruct):
- Qwen2.5-14B-Instruct: 48 layers, 40 q heads, 8 kv heads, head_dim = 128, hidden = 5120, vocab ≈ 152k

Paper §7.5 page 15 textual claim (quote):
> "merely adding embedding noise is insufficient to prevent attackers from recovering tokens via internal states. After applying obfuscation of key matrices, the attacker can hardly exploit the hidden states output by decoder layers to perform ISA. **However, attention scores remain vulnerable to ISA, as the attention scores generated by attention queries and keys are not obfuscated by the key matrices.** Consequently, in AloePri, permutations of attention heads and blocks are introduced to further protect attention scores."

Paper §5.4 page 10 design equation (quote):
> "The attention score for the i-th head is computed as: G(x̃W̃_q^(i))G(x̃W̃_k^η(i))^T ≈_eC G(xW_q^(i'))G(xW_k^η(i'))^T, where i' denotes the permuted index corresponding to the i-th head, and e_C^attn is the obfuscation error induced by block permutation (parameterized by β, γ)."

## Next steps (priority for next session)

### 1. Rigorously challenge the Theorem (P0)

Suggested skill: `/grill-me` or `/grill-with-docs` (project glossary in `CONTEXT.md`).

Specific weak points to attack:

**1a. Lipschitz constant L_A.** Theorem assumes the attack `A` (TTRSR top-1 from ridge inversion + cosine-NN) is L-Lipschitz in its input. Is this actually bounded?
- Ridge inversion `W = (X^T X + αI)^{-1} X^T Y` IS Lipschitz in the input matrix `X` for fixed `α > 0` — bound depends on the conditioning of `X^T X + αI`.
- Cosine-NN top-1 is NOT Lipschitz globally (it's a piecewise-constant argmax). It IS Lipschitz almost-everywhere away from decision boundaries.
- The TTRSR rate (fraction-of-rows-correct) is a sum of indicators; expected value may be Lipschitz under regularity assumptions on the score distribution.
- **Required:** make the Lipschitz argument rigorous, or replace with a probabilistic bound (e.g., total variation distance between plain-vs-obf attack distributions).

**1b. Permutation-invariance assumption.** Theorem assumes `A` is invariant under the head-axis permutation `η` (from inter-head perm τ_kv, τ_group). Holds for flattening attacks; does it hold for ALL attacks?
- Per-head ridge (our `probe3_per_head.py`) is NOT permutation-invariant — it fits a separate inverter per head.
- But per-head ridge is WEAKER than flattened (we measured 11.66% vs 22.41% at L=17). So head perm doesn't help defense even against the head-sensitive attacker.
- **Required:** state the Theorem more carefully — "for the family of head-permutation-invariant attacks" — and separately address per-head attacks.

**1c. Closed-form bound for `e_C^attn(β, γ)`.** Paper gives no closed form. Can we derive one?
- `Ẑ_block` is block-diagonal with each block a w×w permutation, w ∈ [1, β] sampled per the BlockPerm function (Algorithm 2 lines 9-20).
- For `Ẑ_block · Ẑ_block`: if each block-perm `P` is an involution (`P² = I`), then `Ẑ_block² = I` and `e_C^attn = 0`. If random perms, expected `‖Ẑ_block² − I‖ > 0`.
- For β=1, BlockPerm is trivially identity → `e_C^attn = 0` exactly. Our 4B β=1 measurement showed zero security cost vs β=8 — consistent.
- **Required:** derive expected `‖Ẑ_block² − I‖` as a function of β (and γ, the sampling parameter), then compose into `e_C^attn`.

**1d. Edge case: what if obf scores diverge on some heads but match on others?** The Theorem uses `max_i ‖ε_i‖` (Lipschitz under operator norm). Is the bound tight per-head? Are there pathological score distributions where the Lipschitz bound is loose?

**1e. Does the Theorem hold for the **softmax** transformation?** Softmax is L-Lipschitz with `L = 1` on the simplex; we use this to argue post-softmax inherits the bound. But softmax compresses range (we measured ~16pp signal drop). Does this change the *attack's* L_A?
