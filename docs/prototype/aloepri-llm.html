generation
· AloePri

Generative inference without a TEE in the loop. The model is rewritten once, offline, into an obfuscated artifact that any unmodified vLLM / SGLang server can host. The trusted endpoint holds only a secret token-level permutation and the tokenizer — every prompt becomes a string of gibberish on the wire, and the server never learns it isn't English.

§01

Role in the project

where this fits — TODO stub

Generation is the post-retrieval LLM step in private RAG. After the reranker emits its k_max-item AES-GCM bundle, the client decrypts the top-k_final chunks and assembles a prompt for an LLM. AloePri is the route that lets that LLM live on commodity cloud GPUs without a TEE.

What this complements. Embedding, storage, and reranking all run inside a SEV-SNP CVM with a GELO-masked GPU offload (embedding, storage, reranking). Generation has a different cost shape: the model is 5–50× larger than the embedder, and decode is autoregressive, so per-token mask resampling and per-offload PCIe round-trips dominate wall-clock. AloePri sidesteps the cost curve by moving the entire forward pass to an untrusted server and shipping a one-time obfuscated artifact instead.

What's traded. AloePri does not give the per-batch information-theoretic argument GELO provides. Its security is empirical: a static obfuscation, sized so published attacks (VMA, IA, ISA, IMA, NN, TFMA, SDA) recover under 15% of tokens at recommended hyperparameters. The deliberate trade is: accept a weaker but still-meaningful guarantee in exchange for compatibility with unmodified vLLM / SGLang and for scaling validated through 671B (DeepSeek-V3.1-Terminus, per paper).

What's TODO. The downstream integration story — which retrieval shape, which prompt template, which token budget, which deployment topology the generation step sits in — is left as a stub. The protocol surface this page documents is the same regardless of how the surrounding RAG pipeline calls into it.

§02

Definitions & glossary

terms specific to this page
TermMeaning
AloePri Lin et al., Towards Privacy-Preserving LLM Inference via Covariant Obfuscation, arXiv 2603.01499 (ByteDance / Nanjing University). The protocol this page implements.
Covariant obfuscation Joint transform on input data and model weights such that the obfuscated model on obfuscated input produces obfuscated output equivalent to the plaintext pipeline. The "covariant" property is what makes the rewrite lossless: model accuracy is preserved exactly (up to noise terms αe·ε, αh·ε added for VMA defense).
τ (tau) Secret token-level permutation sampled from Sn, where n is vocabulary size (262 144 for Gemma 4). Drawn once per deployment; held by the client; never leaves the trusted zone.
Π (Pi) Permutation matrix corresponding to τ. Applied to the embedding (Π·Wembed*) and model head (Whead*·ΠT) during offline rewrite.
Z Vocabulary mapping Z = { V[i] ↦ V[τ(i)] }. Used online to obfuscate input token IDs and de-obfuscate response token IDs.
P̂, Q̂ (key matrices) Generated by paper Algorithm 1 from secret base matrices (B, B-1, E, F, Z). Satisfy P̂ · Q̂ = Id. P̂ shape is d × (d+2h); Q̂ is (d+2h) × d. Adjacent layers' weights get right-multiplied by some P̂ and left-multiplied by the matching Q̂ so the cancellation runs through every forward pass.
Expansion size h Algorithm 1 hyperparameter. Internal hidden dim grows from d to d+2h inside each transformer block; residual-stream dim stays at d. Default h=128. Paper Figure 5 shows TPOT impact <10% out to h=512 on R1-Distill-14B.
R̂qk, Ĥqk, Ẑblock Paper Algorithm 2 intra-head transforms applied to Q/K weights: 2D rotary obfuscation, 2-block scaling, and RoPE-block-wise permutation respectively. Ẑblock is sampled by BlockPerm under a softmax decay tied to the RoPE base. Restricted to RoPE-rotated dims under p-RoPE.
Ûvo Random invertible matrix applied to V (left of Wv) and inverted on O (right of Wo). Stops V activations from being a clean intermediate the server can attack.
τkv, τgroup Inter-head permutations from Algorithm 2 §5.2.3. τkv shuffles K/V heads; τgroup shuffles Q/O heads within each GQA group. Both are part of the same offline rewrite, with no per-request cost.
αe, αh Gaussian noise coefficients applied to embedding and head weights: W̃embed = Π·(We + αe·ε)·P̂embed. Default αe=1.0, αh=0.2. Paper §7.3: at αe=0.5, VMA jumps to >30% recovery — these knobs matter.
PLE (Per-Layer Embeddings) Gemma 3n / Gemma 4 E-series feature. Table of shape [vocab × 256 × Nlayers] indexed by token_id. The retrieved per-(layer, token) vector is projected to dhidden and added into the residual at each block. Under AloePri the table itself is permuted along the vocabulary axis by τ.
p-RoPE Proportional RoPE. Rotation applied to only the first p · dhead dims (Gemma 4 global layers use p=0.25). Forces R̂qk / Ĥqk / Ẑblock to be identity on the non-rotated subset.
Hybrid attention Gemma 4's interleaved local-sliding-window (W=512 on E2B/E4B) and global attention. Ratio 4:1 (E2B) or 5:1 (E4B). Both layer classes get the same AloePri attention obfuscation, but with separate RoPE-base config (10K local, 1M global).
K=V tying Memory optimisation in Gemma 4 global layers where K and V share a single tensor. AloePri applies different transforms to K (rotary) vs V (Ûvo), so K and V must be stored separately after offline rewrite. τkv still permutes the two in lockstep.
Offline rewrite One-shot client-side transform of plaintext weights into the obfuscated artifact. Reads safetensors, samples τ + all key matrices from a seed, writes safetensors + a .key file holding τ and Z. Per-model, not per-request.
TTRSR Text Token Recovery Success Ratio. Fraction of original tokens an attacker recovers under a given attack. Lower = better privacy. Paper headline: TTRSR <15% on the broadest attack suite at recommended hyperparameters.
VMA / IA / ISA / IMA / NN / TFMA / SDA The seven attacks evaluated against AloePri. VMA / IA exploit plaintext-vs-obfuscated weight relationships; ISA / IMA / NN train inverters from observed internal states; TFMA / SDA exploit token-frequency patterns. The reference codebase ships them at src/security_qwen/.
RmDP Rényi-metric Differential Privacy. Paper §6 definition: combines α-Rényi divergence with a metric on the token-sequence space. The framework used to prove AloePri's noise budget is strictly smaller than a data-only mechanism's at equal accuracy.
§03

Threat model

what each component sees · what it cannot

The trust split here is fundamentally different from the GELO embedder / reranker. There is no hardware-rooted trust anchor in the loop: protection is structural (obfuscation hides the prompt) plus empirical (noise + key matrices defeat published recovery attacks). The trusted endpoint is just the client process that holds τ.

Five empirical leaks the protocol forces a published attack to overcome, ranked by paper-reported attack strength.

Leak channelAttack classMitigationPaper TTRSR ceiling
Plaintext-vs-obfuscated weight pair VMA · IA Embedding / head noise αe·ε, αh·ε; row+column permutation no longer recoverable via sort-then-match < 15% (αe=1.0); leaks > 30% at αe=0.5
Internal-state inversion ISA · IMA · NN Two-side key matrices Q̂ · W · P̂ on every weight; intra-head R̂qk, Ĥqk, Ẑblock, Ûvo; inter-head τkv, τgroup < 15% on hidden states; attention scores get the inter-head permutation on top
Token-frequency statistics TFMA · SDA τ obfuscates token identity at the index level; cross-corpus frequency matching is the residual attack Top-100 ≤ 20% even under distribution-aware adversary; BLEU-4 ≈ 2 on SDA
PLE gather addresses (Gemma 4 only) TFMA variant (novel surface) PLE table permuted along vocab axis by τ; per-(layer, position) gather observes τ(token_id), not the plaintext token ID. Server sees a distribution over obfuscated indices — same TFMA argument, but with Nlayers× the observation volume. Not in paper; measured here. Open risk: TFMA may be more effective on PLE-equipped models than on Qwen-class baselines.
Per-request entropy any future attack exploiting key reuse None at the protocol level — τ, key matrices, and noise ε are sampled once at offline rewrite and reused across every request. The mitigation is the noise + key-matrix construction itself, not freshness. This is the structural difference from GELO's per-batch fresh Haar mask. Accepted trade.

Not covered. Model architecture (the server can fingerprint Gemma 4 from layer count and tensor shapes — only the weights and token identities are obscured). Query rate and timing. Tokenizer roundtrip identity — if a custom tokenizer fails the lossless detok→retok property, the math breaks before security matters. Model-weight privacy — the client needs plaintext weights to perform the offline rewrite; AloePri does not protect the weights themselves from the client.

Supported model architectures

Gemma 4 family — text-generation only at v1

The covariant-obfuscation construction is architecture-generic per paper §8, but every new architectural feature needs its own composition-theorem application. The list below is what's actually validated end-to-end, what's deferred, and what's structurally out of scope.

Model Effective · Raw Layers · Hidden · Head Attention PLE Notes
Gemma 4 E2B 2.3B · 5.1B 35 · 1536 · 256 hybrid 4:1, W=512, RoPE base 10K (local) / 1M (global) 262 144 × 256 × 35 ≈ 1.1 GB int8 Smallest scale; primary validation target.
Gemma 4 E4B (primary) 4.5B · 8B 42 · 2560 · 256 hybrid 5:1, W=512, K=V tied in global layers, p-RoPE p=0.25 in global 262 144 × 256 × 42 ≈ 1.3 GB int8 Headline scale. Exercises every Gemma-4-specific obfuscation deviation (K=V untie, p-RoPE-aware R̂qk, PLE permute).
Gemma 4 31B dense stretch 31B — · — · 256 hybrid 5:1, W=1024 none Tests scaling. Memory ≈ 31 GB int8 fits comfortably in 64 GB host RAM. No PLE.
Gemma 4 26B A4B deferred 26B · 4B active — · — · 256 hybrid 5:1 + MoE 128-of-8, 3× shared expert none Requires CryptoMoE balanced-dispatch defense composed on top of AloePri. Separate research stream.
Multimodal (audio / vision encoders) — — — — Out of scope at v1. Open question on per-modality attack classes (see round-2 §D.9).

Why Gemma 4 specifically. Edge-class deployment target aligns with the project's hardware envelope (commodity Hetzner EPYC + consumer GPU pass-through; no H100-CC dependency). E2B → E4B is a clean 2× scaling axis on a single architecture family. PLE is the most interesting new surface in the post-Qwen LLM landscape — getting it right under AloePri is novel work.

Architectural deviations from the AloePri paper. The paper evaluates Qwen2.5 / Qwen3 / Llama3 / DeepSeek — none of which has PLE, K=V tying, p-RoPE, or hybrid attention. Three Gemma-4-specific extensions to the construction:

Gemma 4 featureAloePri impactResolution
K=V tying in global layers Algorithm 2 applies R̂qk·Ĥqk-1·Ẑblockη to K and Ûvo to V — different transforms, same tensor can't satisfy both Untie K and V into separate tensors during offline rewrite. τkv still permutes them in lockstep. ~16% extra KV-cache memory on global layers (≈ 3% of total).
PLE table on E-series Token-ID-indexed gather per layer creates a per-(layer, position) access-pattern observable; not addressed by the paper Permute the table along the vocab axis: W̃PLE[i, :, ℓ] = WPLE[τ(i), :, ℓ]. Per-layer projection matrix gets right-multiplied by the same P̂ℓ used by the layer-ℓ attention block so the PLE contribution lands in the same obfuscated residual space.
p-RoPE (p=0.25) in global layers R̂qk = block-diag of 2D rotations across all dhead/2 RoPE blocks; only the first 25% of dims are actually rotated R̂qk restricted to the rotated subset; Ĥqk and Ẑblock likewise. Non-rotated subset gets the standard Q̂q / Q̂k left-multiplication only. BlockPerm's rope_base-dependent decay configured separately per local (10K) vs global (1M) layer.
§05

Components

six implementation units · two-sided

Each card pairs [component ↦ source ↦ what it secures ↦ trust side]. Ordered by the lifecycle of a deployment: rewrite the model once → run the server → serve every request through the client wrapper.

offline

obfuscate_gemma4python/aloepri :: rewriter

One-shot CLI that consumes plaintext Gemma 4 safetensors and emits the obfuscated artifact plus a .key file. Samples τ, generates every Algorithm-1 key matrix from a single seed, applies Algorithm 2 per attention layer, applies FFN / RMSNorm / embed / head obfuscation, writes the result. Runs once per (model, seed).

inputs
plaintext config.json + safetensors shards · seed · hyperparameters (λ, h, αe, αh, β, γ)
outputs
obfuscated safetensors with hidden_size = d + 2h · obfuscated config.json · aloepri.key (τ, Z, version tag)
key file
POSIX 0600, never logged, never transmitted. Loss = inability to use any prompt with this artifact.
parity test
For a fixed prompt, logits(plaintext_model, x) ≈ logits(obfuscated_model, Z(x)) · P̂head-1 within eCAloePri; top-1 token agreement ≥ 0.97 on a held-out set.
server

AloePriGemma4ForCausalLMvllm-aloepri :: model

vLLM model class for the obfuscated Gemma 4. Inherits the stock Gemma4ForCausalLM forward graph; differs only in (a) accepting hidden_size = d + 2h from the obfuscated config, (b) routing K and V through separate buffers in global layers, (c) PLE gather + P̂ℓ-aligned projection, (d) p-RoPE-aware attention kernel. Sampling, KV cache, paged attention, P/D split — all unmodified.

registration
ModelRegistry.register("aloepri-gemma4", AloePriGemma4ForCausalLM) at import. config.json's model_type drives selection.
delta-LoC vs stock
~600–900 lines: most of the rewrite is the attention forward and the PLE module. RMSNorm, residual, sampler, scheduler unchanged.
integration risk
vLLM mainline Gemma 4 support is the gating dependency. Fallback path is Gemma 3 (no PLE, no K=V — strictly easier).
client

AloePriClientpython/aloepri :: client

The trusted-side wrapper. Holds τ, Z, and the tokenizer. Every generate(prompt, …) call: tokenize → map IDs through Z → detokenize back to a string → POST that string to vLLM as a normal prompt → tokenize the response string → map through Z-1 → return plaintext.

endpoint
POST {VLLM_URL}/v1/completions with {"prompt": <obfuscated text>, ...} — unmodified vLLM API.
roundtrip property
Asserts tokenizer.encode(tokenizer.decode(ids)) == ids per request as a sanity check; a failure indicates a tokenizer config drift and aborts the call rather than silently corrupting the prompt.
also Rust
Optional thin Rust crate aloepri-client for the existing gelo-rag ingestion path, so the trusted side stays in one language stack.
crypto

KeyMaterialpython/aloepri :: key_material

The full set of secrets generated at offline rewrite. τ is the load-bearing element; the rest derive from a single seed and can be re-generated from it. Persisted as aloepri.key alongside the obfuscated model.

contents
seed (32 B) · τ (S262144, ≈ 1 MB) · Z (precomputed mapping table) · per-layer (B, B-1, E, F, Zorth) bases · ε noise matrices (optional — regenerable from seed)
scope
Lives in the client wrapper's process memory; backed to disk in 0600 mode; never crosses the trust boundary.
rotation
Compromise of the key file = compromise of every past and future prompt under this artifact. Rotation requires re-running the offline rewrite end-to-end and re-uploading the obfuscated model.
eval

attack-harnessevals/aloepri-attacks

The seven-attack benchmark from the AloePri reference codebase, lifted into a standalone harness and wired to consume snapshots from the trusted side of a running deployment. Re-runs published attacks (VMA, IA, ISA, IMA, NN, TFMA, SDA) against Gemma 4 obfuscated artifacts and reports per-attack TTRSR.

shared with
The GELO embedder + reranker use the same harness for their own attack-resistance benches — one corpus, one metric, one CI gate.
release-gate threshold
TTRSR ≤ 15% on each attack at the recommended hyperparameter set; regression vs the previous artifact triggers an alert.
open work
The PLE-aware TFMA variant (see §03) is novel to this deployment and needs new attack code, not just a port.
http

/generatealoepri-client :: api

Optional HTTP front of the client wrapper, for callers that prefer talking to a sidecar over linking a library. Same trust property as the in-process form — the sidecar holds τ, the rest of the application doesn't need to.

request
{ prompt, max_tokens, temperature, top_p, stop, ... } — pass-through to vLLM's sampling params, with the prompt obfuscated en route.
response
{ text, finish_reason, usage } — plaintext, decoded on the trusted side before return.
deployment
Localhost daemon, or an internal-VM sidecar in deployments where τ-holder isolation from the application is a useful operational property.

Compute flow & trust boundaries

per-request — what crosses the wire · per-block — what the server actually computes

Two figures. FIG. 03a traces one full request end-to-end, showing what is plaintext, what is obfuscated, and where the trust boundary sits. FIG. 03b traces one Gemma 4 transformer block on the server side — the entire block is in the untrusted zone, but the model is rewritten such that the server cannot interpret what it is processing.

FIG. 03a — One request through AloePri · plaintext stays inside the trusted zone · only obfuscated text crosses the wire navy = client (trusted) · gpu-tone = vLLM server (untrusted) · dashed amber = obfuscated wire payload

Why the round-trip works. Steps ②③④ on the client and the server-side re-tokenisation in ⑤ are inverses if and only if the tokenizer is round-trip-lossless: tokenizer.encode(tokenizer.decode(ids)) == ids for every ID list the protocol produces. Gemma's SentencePiece is well-behaved on standard text, but byte-fallback tokens and special tokens (<bos>, <eos>, image / audio placeholders) need explicit fuzz coverage — the client wrapper asserts the property per request as a guard against silent corruption from a tokenizer-config drift.

FIG. 03b — One Gemma 4 transformer block on the server · same shape as plaintext Gemma 4 · every weight already absorbed an Algorithm-1 / Algorithm-2 transform orange band = untrusted zone (entire block) · the block runs on commodity GPU with no protocol awareness

What to look for between this and a stock Gemma 4 block. ① Same topology: pre-LN RMSNorm, GQA attention, SwiGLU FFN, residual. vLLM's stock forward graph runs unchanged. ② Every weight tensor differs: each weight has absorbed an Algorithm-1 left-multiplied Q̂ and a right-multiplied P̂ from the matching base set. The cancellation runs through the entire forward pass — Q̂q·P̂previous_layer = Id means the obfuscation contributions to the inputs land back on the unit when they hit the next Q̂. ③ Three Gemma-specific deltas: the PLE branch on E-series, the K=V untie + separate storage in global layers, and the p-RoPE-restricted R̂qk / Ĥqk / Ẑblock. ④ No new ops: there are no AloePri-specific kernels, no protocol-aware attention variant, no custom sampler. The entire scheme is "different numbers in the same matmuls."

Where the obfuscation actually lives

Layer / opPlaintext formObfuscated formComposition argument
Embedding We[x, :] (Π · (We + αe·ε) · P̂embed)[τ(x), :] Π row-permutes; P̂embed right-multiplies so output lands in P̂embed space; noise defeats VMA
RMSNorm x · γ / ‖x‖RMS Covariant via paper §5.2.4: γ absorbs the Q̂ acting on the right-multiplication of the previous layer; norm is invariant under right-orthogonal action when applied within obfuscated space Norm-preserving by construction; absorbing γ into the obfuscated weight chain is the point of the covariant property
Q / K / V projections x · Wq,k,v See Algorithm 2 §5.2.3 — intra-head R̂qk, Ĥqk, Ẑblock, Ûvo applied; inter-head τkv, τgroup permute heads R̂qk is a rotation in RoPE-block space — commutes with RoPE; Ẑblock is a permutation of RoPE 2-blocks — softmax / dot-product invariant since both Q and K get permuted identically
Output projection attn · Wo (attn · Ûvo-1) · Wo · P̂o Ûvo cancels the V-side; P̂o lifts back into the layer's obfuscated residual space
FFN (gate / up / down) standard SwiGLU Each weight wrapped Q̂ · W · P̂; SiLU and Hadamard product commute with the elementwise key-matrix structure since Q̂ acts on the d-axis Sequential composition theorem (paper §4.2.1) chains the Q̂ at gate / up's input with the P̂ at down's output; intermediate inflated dim absorbs the expansion h
PLE gather (E-series) WPLE[tok, :, ℓ] (WPLE[τ-1(τ(tok)), :, ℓ]) · W̃PLE_projℓ = same value, projected via P̂ℓ Vocabulary-axis permutation is invariant under gather since the lookup key is permuted in lockstep; per-layer projection's right-mult P̂ℓ matches the residual-stream key
LM head Hfinal · Wh (H̃final · Q̂head) · (Wh + αh·ε) · ΠT Q̂head cancels the residual stream's P̂; ΠT obfuscates the output-vocabulary identity so the sampler picks τ(true_next_token); noise αh·ε defeats head-side VMA

The composition theorems in paper §4.2 (sequential, parallel, summation) are what make this scheme work at scale: any DAG built from covariant-obfuscated nodes is itself covariant-obfuscated, with the obfuscation error bounded by the sum of per-component errors. That's why adding new layer types (PLE, hybrid attention, p-RoPE) is a per-component obfuscation derivation rather than a re-design of the whole protocol — each one is a leaf that the composition theorems lift into the full forward.
