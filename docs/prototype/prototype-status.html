§01

What this prototype is

scope · approach 4

A working implementation of Approach 4 from docs/research/private-rag-system-design.md: thin-client RAG where embedding happens inside an attested SEV-SNP CVM, the heavy matmuls offload to an information-theoretically blind GPU, and vectors live in storage as distance-preserving ciphertext. A parallel service implements Recipe C / RemoteRAG — DP-perturbed query, Paillier-reranked, plaintext-doc-index — for deployments where query confidentiality is the headline threat.

The prototype answers a specific question: can openweight embedding inference be served by an untrusted cloud GPU at near-plaintext cost without the GPU ever learning the input? The literature offers three families of answers — full FHE (orders of magnitude too slow), confidential-compute GPUs (10× cost premium, vendor lock-in), and TEE-on-CPU + masked-GPU-offload (GELO, TwinShield, SCX). This codebase ships the third family.

The repository is in a pre-product state. Protocol math, attestation byte format, mock issuer/verifier, in-process simulator, BEIR/NFCorpus accuracy & throughput, and the systemd-bootable HTTP runner are all green at tiers T1 + T2. Real-silicon T3 (Hetzner EPYC Genoa + VFIO GPU passthrough) is deferred to milestone M5.9.

CAPRISE-in-TEE wiring (Option 3 / two-party HKDF) — delivered 2026-05-14. The runner derives the CAPRISE key per request from HKDF(user_x_sk ‖ tee_user_x_sk, salt = tenant_id), zeroizes on return, fails loudly with HTTP 410 when the TEE half-secret is missing (CVM restarted). Forward-secure against TEE-only compromise; full plan and threat-model walkthrough in docs/prototype/caprise-two-party-kdf.md. See §07 for the M1–M6 delivery table.

crypto / at rest

CAPRISErag_core :: caprise

Distance-preserving embedding encryption — search the ciphertext directly with plain cosine, recover the cleartext only with the key.

method
scale + radial-Gaussian-noise on unit sphere; PRF-derived per-nonce direction. Asymmetric coefficients: 3/8 docs, 1/8 query — preserves rank order under cosine.
source
Ye et al. 2026 / prRAG · arXiv. Spec at caprise.rs.
target
stored embedding vectors; the index server learns no useful information about either the documents or which one matched.
threat
curious storage operator with full DB access but no CapriseKey. Out of scope: a key-holder running Vec2Text — that's what DP-Forward catches.
cost
~0 retrieval overhead · 2,339 vec/s encryption · Vec2Text BLEU drops 83 → 12.4 (paper).

method

source
in-house port of the SAP / DCPE family. Spec at sap.rs.
target
same as CAPRISE: stored embeddings at rest.
threat
same as CAPRISE. Retired in favour of CAPRISE in beir_accuracy.rs; lives on for round-trip parity tests.
cost

AES-GCM chunksrag_core :: content

AES-256-GCM authenticated encryption of document chunk text. Independent of embedding encryption — both layers protect the same row.

method
32-byte client key, fresh 96-bit nonce per chunk, GCM authenticator.
source
aes-gcm crate 0.10.3 (RustCrypto).
target
raw chunk text returned alongside top-k embeddings. Decrypted only on the client (or inside the TEE on reranking paths).
threat

cost

TEE / split inference

GELO mask gelo_protocol :: mask

Per-batch Haar-uniform orthogonal matrix A, applied on the token axis: U=A·H ships to the GPU, H·W = Aᵀ·(U·W) recovers on return.

method
Householder QR over a fresh ChaCha20Rng Gaussian seed; bit-exact recovery (modulo FP cancellation). Optional shield rows close the multi-batch Gram leak (U⊤U = H⊤H).
source
Belikov & Fedotov, GELO — arXiv 2603.05035. Shield rows from TwinShield (Xue et al. 2025 §V-B).
target
activations crossing PCIe to the untrusted GPU; weights stay public, only H is hidden.
threat
untrusted GPU + driver + host. Survives the 2026 precomputed-basis attack (arXiv 2602.11088) because A is full-rank Haar, not a static basis.
cost
+6.4% wall vs gpu_plain on Qwen3 at embedding shape (3-text micro-bench). 17 ms / text of irreducible mask machinery across 168 offloaded linears.
TEE / hardening

Shield rowsgelo_protocol :: shield

k random high-energy rows spliced into H before masking and stripped after recovery — turns U⊤U = H⊤H into H⊤H + S⊤S so multi-batch BSS / ICA cannot isolate the cleartext Gram.

method
Default k=8 shield rows at energy ≈ 4·mean‖h‖ (production paper-parity setting). Stack into scratch, mask, recover, slice off.
source
TwinShield (Xue et al. 2025 §V-B). Empirically tested in tests/bss_recovery.rs — FastICA recovery succeeds without shield, fails with.
target
cross-batch Gram leak in the GELO construction.
threat
persistent GPU-side observer collecting many U_i for an ICA-style recovery.
cost
+28 ms / text (stack) + 14 ms (strip) across 112 calls per Qwen3 forward.
TEE / integrity

U-Verifygelo_protocol :: integrity

Freivalds-style integrity probe: for asserted Z = A·B the TEE checks A·(B·r) ≈ Z·r against a random r — undetected-tamper rate ≤ (2L)⁻ᵏ.

method
Random r ∈ {-L..L}ᵖ, k independent probes. Ships L=3; deployment picks k. TEE keeps a B-side reference (Arc-shared with the embedder's weights → no duplicate 2.4 GB cache on Qwen3).
source
Freivalds (1979) / TwinShield §V-C operationalisation.
target
integrity of every offloaded GEMM; a malicious GPU returning crafted garbage is rejected.
threat
actively-malicious untrusted GPU.
cost
k=2 ⇒ ≈ 2.8% miss · k=8 ⇒ 2.4·10⁻⁷ miss. Disabled by default in the throughput bench (k=0); on for adversarial settings.
TEE / attention

OutAttnMultgelo_protocol :: out_attn_mult

4-partition embedding for Q·Kᵀ — both operands are runtime values, neither side can be pre-uploaded. GPU sees a (2n,2n) masked permuted matmul without the recovery scalars.

method
Mask Q + R_Q, Kᵀ + R_Kt with fresh random matrices and scale by random scalars a, b; stack into (2n, d)·(d, 2n) with row/column permutations; TEE recovers four partitions into Q·Kᵀ.
source
TwinShield (Xue et al. 2025 §V-A).
target
the one matmul GELO doesn't cover (no public weight). Auto-switches off below n ≈ hidden_size — at embedding shape n ≪ d, the 4× FLOP widening loses to in-TEE attention.
threat
same as GELO: untrusted GPU sees Q/K only in the 4-partition form without the secret scalars.
cost
+24% wall when forced on at embedding shape (regime mismatch); net win at long context (n in thousands).
TEE / research lever

Permuted attentiongelo_protocol :: attention

Amulet-inspired softmax-equivariance: softmax(πQKᵀπᵀ/√d)·πV = π·softmax(QKᵀ/√d)·V — fresh row-permutation lets softmax + both attn matmuls run on the GPU under obfuscation.

method
Per-batch π ∈ S_n, masked-causal mask transformed accordingly, optional σ ≈ 0.01 Gaussian noise (Hidden No More mitigation, arXiv 2505.18332).
source
Amulet — Wang et al., arXiv 2512.07495. SCX-style stateless mask derivation deferred.
target
the in-TEE-attention bottleneck (28% of Qwen3 per-text wall at n≈400).
threat
same model as GELO. Survives arXiv 2505.18332 only when paired with σ-noise + shield.
cost
regresses at embedding shape — 84 extra dispatches/text on iGPU ⇒ 300 ms vs 153 ms. Defaults off; opt-in via BEIR_PERM_ATTN=1. Breakeven n ≈ 1000–2000 on this hardware.
crypto / privacy

DP-Forwarddp_forward

Analytic Matrix Gaussian Mechanism on the pooled embedding — irreversible Gaussian noise giving formal (ε, δ)-Sequence-LDP. Survives any downstream encryption.

method
Per-row L2 clip to C; Balle–Wang analytic-Gaussian σ at fixed (ε, δ, Δ=2C); isotropic N(0, σ²I) noise on the pooled output. Applied inside the embedder so the SEV-SNP report binds to (ε, δ, C, σ).
source
Yue et al., DP-Forward — arXiv 2309.06746, CCS 2023.
target
the residual after a key-holder decrypts the CAPRISE index. The only thing that ever leaves the embedder is e + DP_noise — CAPRISE-keyed inversion still hits the DP wall.
threat
compromised CAPRISE key (TEE breach, legal compulsion, key-rotation bug). Out of scope: anything before the embedder.
cost
µs per pooled vector. Empirically (pooled, ε=4) destroys NFCorpus retrieval — kept available, not on by default; documented finding in docs/prototype/dp-forward.md §6.
crypto / privacy

Planar-Laplaceremote_rag :: planar_laplace

Query-side (n, ε)-DistanceDP — perturb q to q' = q + r·v with r ~ Gamma(n, 1/ε) and v uniform on S^{n−1}.

method
Gamma-distributed radial magnitude, Gaussian-normalised direction. n is the embedding dimension, not a cluster count — calibration is ε ∼ 10n…50n.
source
RemoteRAG — Cheng et al., ACL Findings 2025 (arXiv 2412.12775).
target
the query operand on its way to a plaintext-index server. Stage 2 (Paillier rerank) recovers exact ranking under the clean q.
threat
untrusted retrieval server. Mutually exclusive with CAPRISE-at-rest — see Paillier card.
cost
µs per query; the latency budget for RemoteRAG lives in Paillier (Stage 2).
crypto / privacy

Paillier rerankremote_rag :: paillier

Additively-homomorphic dot-product: ∏ᵢ Enc(qᵢ)^{e_d[i]_int} = Enc(⟨q, e_d⟩). Stage-2 of RemoteRAG — reranks candidate docs against the clean query without revealing it.

method
Custom Paillier on num-bigint (pure-Rust, no GMP). Full CRT factor precompute; fixed-point quantisation; signed-result decode. Hot path on rayon.
source
RemoteRAG (arXiv 2412.12775); algorithms adapted from fast-paillier 0.3.2 (LFDT-Lockness, Apache/MIT). Pure-Rust backend, no LGPL footgun.
target
exact query–document cosine on a plaintext-index server without revealing the query.
threat
untrusted retrieval server. Cannot coexist with CAPRISE-at-rest — Paillier's additive HE requires plaintext doc embeddings as exponents.
cost
~750 ms CRT-encrypt + ~47 ms × k′ rerank at 256-bit; HNSW Stage-1 at 10k docs ⇒ 23.5 ms end-to-end mean.

Production CVM evidence path. 1,184-byte SEV-SNP attestation report with REPORT_DATA binding (model_identity, scheme_identity, nonce), signed by the per-chip VCEK in the AMD-SP.

cost

orchestrator / TEE

GeloRagTwoPartyServicegelo_rag :: two_party_service

Multi-tenant CAPRISE-in-TEE service — holds no per-tenant scheme at idle. Every ingest_chunks_for / query_for request derives the CAPRISE key per call from HKDF(user_x_sk ‖ tee_user_x_sk, salt = tenant_id) and zeroizes it at return. The shape gelo-snp-runner wires.

method
Per-tenant tee_user_x_sk + index map. Client sends (tenant_id, user_x_sk) over RATLS per request; CVM looks up its half-secret, derives (caprise_seed, aes_chunk_key) via the keying module, encrypts/decrypts inside the call, drops everything. Missing tenant → TwoPartyError::UnknownTenant → HTTP 410 Gone.
source
in-house. two_party_service.rs. Spec: caprise-two-party-kdf.md. Realises design-doc Option 3 (two-party HKDF).
target
forward-security against TEE-only compromise: a future seal break recovers tee_user_x_sk but not user_x_sk (never persisted; the TLS that carried it is ECDHE-forward-secret) — past sessions remain undecryptable.
threat
untrusted cloud operator (TEE memory dump, CVM rollback, persistent-disk reads). Breaks only under simultaneous TEE+client compromise.
cost
one HKDF-Extract + two HKDF-Expand per request (µs-scale) plus the existing embedder + CAPRISE costs. Variant A persistence (in-memory only) today; Variants B/C deferred to M7.
crypto / KDF

HkdfPolicyrag_core :: keying

The two-party KDF type. One Extract + two Expands over RustCrypto's hkdf 0.12 + hmac + sha2 — no hand-rolled HMAC, no hand-rolled PRF chaining.

method
derive(user_x_sk, tee_user_x_sk, tenant_id) → (caprise_seed, aes_chunk_key). Both outputs are Zeroizing<[u8; 32]>. Versioned info strings (gelo-rag.v1.caprise.seed, gelo-rag.v1.aes-chunks) — bumping to v2 produces entirely fresh keys.
source
in-house. core/src/keying.rs. 6 unit tests pin determinism + cross-tenant salt separation + scheme_identity stability.
target
cross-session deterministic re-derivation (same inputs ⇒ same keys) so ingest and query share a key without persisting it between them. Tenant id is the HKDF salt ⇒ cross-tenant key separation by construction.
threat
downgrade between versions — pin the policy in scheme_identity (REPORT_DATA) and reject mismatches at handshake.
cost
µs per call. scheme_identity_digest() exposes the canonical SHA-256 of (policy ‖ CAPRISE constants) for attestation binding.

Interfaces

crate seams · trait surface

Three trait pairs define every seam in the system. The split is deliberately narrow — the GPU engine knows nothing about masking, the trusted executor knows nothing about retrieval, the retrieval service knows nothing about attestation issuance.

  • provision_weight(handle, W) — public weight upload at model load (once)
  • begin_forward_pass(n) / end_forward_pass — paper-parity mask lifecycle

  • offload_qkv(layer, H) → (Q, K, V) — shared-mask 3-way
  • offload_linear_many(handles, H) — SwiGLU gate+up bundled
  • offload_attention_qkt(Q, Kᵀ) → Q·Kᵀ — OutAttnMult path
  • offload_attention_permuted(Q, K, V, scale, mask) — Amulet-style

GeloRagTwoPartyService — what the runner binds (CAPRISE-in-TEE)

  • new(embedder, verifier) · with_params(embedder, verifier, scheme_params, hkdf_policy) — generic over each seam, no scheme field (derived per request)

Highest-impact open levers

  • Real T3 boot — Hetzner EPYC provisioning + one captured VCEK + sample report for offline CI
  • GPU-side OutAttnMult stacking — fused WGSL kernel for 2n-wide operand packing; long-context win
  • TDISP for DMA — removes SWIOTLB bounce (~15 ms/Qwen3-text) once kernel + firmware ships
  • SCX stateless mask derivation — HKDF-keyed per (session, request, layer, op); unlocks horizontal scaling + mask audit
  • Amulet softmax-equivariant attention — research lever; pays off at long context, regresses at embedding shape

Out of scope (explicitly)

  • Private model weights — would need STIP-style weight masking; whole stack is built on the openweight assumption
  • FHE / MPC inference — orders of magnitude too slow for the embedding workload
  • Side-channels in the TEE — SEV-SNP isolates memory but not cache / timing / power; subtle constant-time hardening deferred
  • Hosted generation — out of scope for GeloRAG; client-side LLM expected per the design doc
  • Hybrid BM25 + dense retrieval — open research gap; nothing private ships it today
